Ruby 1.8 へ sqlite3 を入れる手順

インストール
>gem install sqlite3 -v  "1.3.3"

動作確認
>ruby -r rubygems -e 'require "sqlite3"'

gem_original_require': no such file to load -- sqlite3/sqlite3_native (LoadError)
のエラーが出る

gem のパスを確認
>ruby -r rubygems -e 'require "sqlite3"'


対処方法１
lib\sqlite3 へ 配下の 1.8 ディレクトリ の sqlite3_native.so を一つ上の階層へ移動する

対処方法２
gemspec の lib のパスを  s.require_paths=["lib"] から s.require_paths= ["lib/sqlite3_native"] に変更するらしい （未確認）

動作確認
>ruby -r rubygems -e 'require "sqlite3"'


C:/Ruby187/lib/ruby/gems/1.8/gems/sqlite3-1.3.3-x86-mingw32/lib/sqlite3/sqlite3_native.so: 193: %1 is not a valid Win32 application.
   - C:/Ruby187/lib/ruby/gems/1.8/gems/sqlite3-1.3.3-x86-mingw32/lib/sqlite3/sqlite3_native.so (LoadError)
のエラーが出る


PATH が通っている場所の sqlite3.dll を 32bit版に変更する

動作確認
>ruby -r rubygems -e 'require "sqlite3"'
